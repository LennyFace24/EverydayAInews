## 问题分析

用户在使用订阅表单时收到错误信息：`"The \`email\` is not a valid email address."`，状态码为 422。

### 根本原因
1. **缺乏前端验证**：前端订阅表单没有对电子邮件格式进行验证，直接发送用户输入
2. **缺乏后端验证**：后端 API 没有在调用 Resend API 前验证电子邮件格式
3. **错误处理不完善**：当 Resend API 返回验证错误时，后端没有正确处理并返回友好的错误信息

## 修复方案

### 1. 前端验证改进
- 在 `app/page.tsx` 中添加电子邮件格式验证
- 使用正则表达式验证电子邮件格式
- 在提交前检查输入是否为空和格式是否正确
- 显示友好的验证错误提示

### 2. 后端验证改进
- 在 `app/api/subscribe/route.ts` 中添加服务器端电子邮件验证
- 使用正则表达式或专门的验证库验证电子邮件格式
- 在调用 Resend API 前进行验证，避免无效请求

### 3. 错误处理优化
- 改进后端错误处理逻辑，捕获并区分不同类型的错误
- 当 Resend API 返回验证错误时，返回 400 状态码和友好的错误信息
- 保持其他服务器错误返回 500 状态码

## 预期效果

- 用户在输入无效电子邮件格式时会收到即时的前端验证提示
- 即使绕过前端验证，后端也会拒绝无效的电子邮件
- 所有错误都会返回友好的错误信息，提高用户体验
- 减少对 Resend API 的无效请求，提高系统效率
